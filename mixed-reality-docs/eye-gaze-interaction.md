## <a name="eye-gaze-input-design-guidelines"></a><span data-ttu-id="c4653-121">目视输入设计准则</span><span class="sxs-lookup"><span data-stu-id="c4653-121">Eye-gaze input design guidelines</span></span>

### <a name="benefits-of-eye-gaze-input"></a><span data-ttu-id="c4653-124">目视输入的优点</span><span class="sxs-lookup"><span data-stu-id="c4653-124">Benefits of eye-gaze input</span></span>
- <span data-ttu-id="c4653-125">**高速指向。**</span><span class="sxs-lookup"><span data-stu-id="c4653-125">**High speed pointing.**</span></span> <span data-ttu-id="c4653-126">眼睛判断力是人为身体最快的响应判断力。</span><span class="sxs-lookup"><span data-stu-id="c4653-126">The eye muscle is the fastest reacting muscle in the human body.</span></span> 

- <span data-ttu-id="c4653-127">**不费力。**</span><span class="sxs-lookup"><span data-stu-id="c4653-127">**Low effort.**</span></span> <span data-ttu-id="c4653-128">几乎没有任何身体动作。</span><span class="sxs-lookup"><span data-stu-id="c4653-128">Barely any physical movements are necessary.</span></span> 

- <span data-ttu-id="c4653-129">**隐含性。**</span><span class="sxs-lookup"><span data-stu-id="c4653-129">**Implicitness.**</span></span> <span data-ttu-id="c4653-130">用户通常会将其描述为 "构思阅读"，有关用户的目视变动的信息使系统可以知道用户打算参与哪个目标。</span><span class="sxs-lookup"><span data-stu-id="c4653-130">Often described by users as "mind reading", information about a user's eye movements lets the system know which target the user plans to engage.</span></span> 

- <span data-ttu-id="c4653-131">**备选的输入通道。**</span><span class="sxs-lookup"><span data-stu-id="c4653-131">**Alternative input channel.**</span></span> <span data-ttu-id="c4653-132">眼睛可为用户提供强大的支持输入，并根据用户的手眼协调从用户的经验中生成。</span><span class="sxs-lookup"><span data-stu-id="c4653-132">Eye-gaze can provide a powerful supporting input for hand and voice input building on years of experience from users based on their hand-eye coordination.</span></span>

- <span data-ttu-id="c4653-133">**视觉注意力。**</span><span class="sxs-lookup"><span data-stu-id="c4653-133">**Visual attention.**</span></span> <span data-ttu-id="c4653-134">另一个重要的优点是可以推断出用户正在关注的内容。</span><span class="sxs-lookup"><span data-stu-id="c4653-134">Another important benefit is the possibility to infer what a user is paying attention to.</span></span> <span data-ttu-id="c4653-135">这可以帮助不同的应用程序领域，从更有效地评估不同设计，到协助的用户界面，增加了社交提示以进行远程通信。</span><span class="sxs-lookup"><span data-stu-id="c4653-135">This can help in various application areas ranging from more effectively evaluating different designs to aiding in smarter user interfaces and enhanced social cues for remote communication.</span></span>

<span data-ttu-id="c4653-136">简而言之，使用眼睛眼睛作为输入可提供快速且简单的上下文输入信号。</span><span class="sxs-lookup"><span data-stu-id="c4653-136">In a nutshell, using eye-gaze as an input offers a fast and effortless contextual input signal.</span></span> <span data-ttu-id="c4653-137">与其他输入（如*语音*和*手动*输入）相结合时，此功能特别强大，可以确认用户的意图。</span><span class="sxs-lookup"><span data-stu-id="c4653-137">This is particularly powerful when combined with other inputs such as *voice* and *manual* input to confirm the user's intent.</span></span>


### <a name="challenges-of-eye-gaze-as-an-input"></a><span data-ttu-id="c4653-138">眼睛为输入的挑战</span><span class="sxs-lookup"><span data-stu-id="c4653-138">Challenges of eye-gaze as an input</span></span>
<span data-ttu-id="c4653-139">如果有大量的强大功能，就有了许多责任。</span><span class="sxs-lookup"><span data-stu-id="c4653-139">With lots of power, comes lots of responsibility.</span></span>
<span data-ttu-id="c4653-140">虽然眼睛可用于创建符合用户体验的用户体验，使你感觉像是 superhero，但了解这一点也很重要。</span><span class="sxs-lookup"><span data-stu-id="c4653-140">While eye-gaze can be used to create satisfying user experiences which make you feel like a superhero, it is also important to know what it is not good at to appropriately account for this.</span></span> <span data-ttu-id="c4653-141">下面讨论了一些需要考虑的问题以及如何解决这些*问题*：</span><span class="sxs-lookup"><span data-stu-id="c4653-141">The following discusses some *challenges* to consider and how to address them when working with eye-gaze input:</span></span> 

- <span data-ttu-id="c4653-142">**眼睛为 "始终打开"** 打开眼睛护盖后，眼睛开始 fixating 环境中的东西。</span><span class="sxs-lookup"><span data-stu-id="c4653-142">**Your eye-gaze is "always on"** The moment you open your eye lids, your eyes start fixating on things in the environment.</span></span> <span data-ttu-id="c4653-143">对每个外观作出反应并意外发出操作，因为您查看的内容太长，会导致 unsatisfying 的体验。</span><span class="sxs-lookup"><span data-stu-id="c4653-143">Reacting to every look you make and accidentally issuing actions, because you looked at something for too long, would result in an unsatisfying experience.</span></span>
<span data-ttu-id="c4653-144">因此，建议将眼睛与*声音命令*、手势、*按钮单击*或扩展停留结合*起来，以*触发选择目标（有关详细信息，请参阅[眼睛和提交](gaze-and-commit-eyes.md)）。</span><span class="sxs-lookup"><span data-stu-id="c4653-144">Therefore, we recommend combining eye-gaze with a *voice command*, *hand gesture*, *button click* or extended dwell to trigger the selection of a target (for more information see [eye-gaze and commit](gaze-and-commit-eyes.md)).</span></span>
<span data-ttu-id="c4653-145">此解决方案还允许使用一种模式，在该模式下，用户可以自由地查找，而不会因触发某些事情而 involuntarily。</span><span class="sxs-lookup"><span data-stu-id="c4653-145">This solution also allows for a mode in which the user can freely look around without being overwhelmed by involuntarily triggering something.</span></span> <span data-ttu-id="c4653-146">当查看目标时设计视觉对象和听觉反馈时，还应考虑此问题。</span><span class="sxs-lookup"><span data-stu-id="c4653-146">This issue should also be considered when designing visual and auditory feedback when looking at a target.</span></span>
<span data-ttu-id="c4653-147">尝试不要使用户塞满即时弹出效果或悬停声音。</span><span class="sxs-lookup"><span data-stu-id="c4653-147">Try not to overwhelm the user with immediate pop-out effects or hover sounds.</span></span> <span data-ttu-id="c4653-148">个很微妙为 key。</span><span class="sxs-lookup"><span data-stu-id="c4653-148">Subtlety is key.</span></span> <span data-ttu-id="c4653-149">讨论[设计建议](eye-gaze-interaction.md#design-recommendations)时，我们将在下面讨论一些最佳实践。</span><span class="sxs-lookup"><span data-stu-id="c4653-149">We will discuss some best practices for this further below when talking about [design recommendations](eye-gaze-interaction.md#design-recommendations).</span></span>

- <span data-ttu-id="c4653-150">**观察与控制**假设您想要在墙壁上精确地伸直照片。</span><span class="sxs-lookup"><span data-stu-id="c4653-150">**Observation vs. control** Imagine that you want to precisely straighten a photograph on your wall.</span></span> <span data-ttu-id="c4653-151">你会参照它的边框和四周，检查它是否平齐。</span><span class="sxs-lookup"><span data-stu-id="c4653-151">You look at its borders and its surroundings to see if it aligns well.</span></span> <span data-ttu-id="c4653-152">现在，如果您想要使用眼睛作为输入来移动图片，则可以想象出如何实现此目的。</span><span class="sxs-lookup"><span data-stu-id="c4653-152">Now imagine how you would do that when you want to use your eye-gaze as an input to move the picture.</span></span> <span data-ttu-id="c4653-153">有点难度，对不对？</span><span class="sxs-lookup"><span data-stu-id="c4653-153">Difficult, isn't it?</span></span> <span data-ttu-id="c4653-154">这介绍了输入和控制需要时，眼睛的双重角色。</span><span class="sxs-lookup"><span data-stu-id="c4653-154">This describes the double role of eye-gaze when it is required both for input and control.</span></span> 

- <span data-ttu-id="c4653-155">**在单击前保留：** 对于快速目标选择，研究表明，用户的眼睛可以在结束手动单击（例如，单击）前继续。</span><span class="sxs-lookup"><span data-stu-id="c4653-155">**Leave before click:** For quick target selections, research has shown that a user's eye-gaze can move on before concluding a manual click (e.g., an air tap).</span></span> <span data-ttu-id="c4653-156">因此，必须特别注意的是，将快速目视眼睛的信号与慢速控制输入（例如语音、双手、控制器）进行同步。</span><span class="sxs-lookup"><span data-stu-id="c4653-156">Hence, special attention must be paid to synchronizing the fast eye-gaze signal with slower control input (e.g., voice, hands, controller).</span></span>

- <span data-ttu-id="c4653-157">**小型目标：** 当您尝试读取只是太小而无法阅读的文本时，您是否知道这种感觉？</span><span class="sxs-lookup"><span data-stu-id="c4653-157">**Small targets:** Do you know the feeling when you try to read text that is just a bit too small to read comfortably?</span></span> <span data-ttu-id="c4653-158">这种令人厌烦的感觉会使您感到厌倦和磨损，因为您尝试重新调整您的眼睛，使其更好地专注。</span><span class="sxs-lookup"><span data-stu-id="c4653-158">This straining feeling on your eyes can cause you to feel tired and worn out, because you try to readjust your eyes to focus better.</span></span>
<span data-ttu-id="c4653-159">当你强制用户使用目视目标选择在你的应用程序中太小的目标时，你可以在用户中调用这种感觉。</span><span class="sxs-lookup"><span data-stu-id="c4653-159">This is a feeling you might invoke in your users when forcing them to select targets that are too small in your application using eye targeting.</span></span>
<span data-ttu-id="c4653-160">在设计方面，为了给用户建立一种愉悦舒适的体验，我们建议目标至少在 2° 的视角范围内，最好是再大一些。</span><span class="sxs-lookup"><span data-stu-id="c4653-160">For your design, to create a pleasant and comfortable experience for your users, we recommend that targets should be at least 2° in visual angle, preferably larger.</span></span>

- <span data-ttu-id="c4653-161">**眼睛不规则的运动**我们的眼睛执行从固定到固定的快速移动。</span><span class="sxs-lookup"><span data-stu-id="c4653-161">**Ragged eye-gaze movements** Our eyes perform rapid movements from fixation to fixation.</span></span> <span data-ttu-id="c4653-162">观察录制的眼部运动扫描路径时你会发现，这些路径看起来是不规则的。</span><span class="sxs-lookup"><span data-stu-id="c4653-162">If you look at scan paths of recorded eye movements, you can see that they look ragged.</span></span> <span data-ttu-id="c4653-163">您的眼睛会在与*打印头*或*手间运动*比较的情况中快速跳转。</span><span class="sxs-lookup"><span data-stu-id="c4653-163">Your eyes move quickly and in spontaneous jumps in comparison to *head-gaze* or *hand motions*.</span></span>  

- <span data-ttu-id="c4653-164">**跟踪可靠性：** 当眼睛调整到新的条件时，眼睛跟踪准确性可能会降低变化的光线。</span><span class="sxs-lookup"><span data-stu-id="c4653-164">**Tracking reliability:** Eye tracking accuracy may degrade a little in changing light as your eyes adjust to the new conditions.</span></span>
<span data-ttu-id="c4653-165">虽然这应该不会影响应用程序的设计，但准确性应在2°限制范围内，因此用户可能需要重新校准。</span><span class="sxs-lookup"><span data-stu-id="c4653-165">While this should not necessarily affect your application design, as the accuracy should be within the 2° limitation, it may be necessary for the user to calibrate again.</span></span> 


## <a name="design-recommendations"></a><span data-ttu-id="c4653-166">设计建议</span><span class="sxs-lookup"><span data-stu-id="c4653-166">Design recommendations</span></span>
<span data-ttu-id="c4653-167">下面是基于目视输入的所述优点和挑战的特定设计建议列表：</span><span class="sxs-lookup"><span data-stu-id="c4653-167">The following is a list of specific design recommendations based on the described advantages and challenges for eye-gaze input:</span></span>

1. <span data-ttu-id="c4653-168">**眼睛看不到打印头：**</span><span class="sxs-lookup"><span data-stu-id="c4653-168">**Eye-gaze is not the same as Head-gaze:**</span></span>
    - <span data-ttu-id="c4653-169">**考虑快速但不规则的眼睛是否符合输入任务：** 虽然我们的快速而引人注目的变动非常适合于在我们的视图中快速选择目标，但它不适用于需要平滑输入轨迹（如绘图或 encircling 批注）的任务。</span><span class="sxs-lookup"><span data-stu-id="c4653-169">**Consider whether fast yet ragged eye movements fit your input task:** While our fast and ragged eye movements are great at quickly selecting targets across our field of view, it is less applicable for tasks that require smooth input trajectories (e.g., drawing or encircling annotations).</span></span> <span data-ttu-id="c4653-170">在这种情况下，应该首选手部或头部指向。</span><span class="sxs-lookup"><span data-stu-id="c4653-170">In this case, hand or head pointing should be preferred.</span></span>
  
    - <span data-ttu-id="c4653-171">**避免直接将某些内容附加到用户的眼睛，如滑块或光标。**</span><span class="sxs-lookup"><span data-stu-id="c4653-171">**Avoid attaching something directly to the user’s eye-gaze (e.g., a slider or cursor).**</span></span>
<span data-ttu-id="c4653-172">如果是游标，这可能会导致 "fleeing cursor" 效果，原因是预计的眼睛眼睛略有偏差。</span><span class="sxs-lookup"><span data-stu-id="c4653-172">In case of a cursor, this may result in a "fleeing cursor" effect due to slight offsets in the projected eye-gaze signal.</span></span> <span data-ttu-id="c4653-173">如果是滑块，则它可能与用眼睛控制滑块的双角色冲突，同时也需要检查对象是否位于正确的位置。</span><span class="sxs-lookup"><span data-stu-id="c4653-173">In case of a slider, it can conflict with the double role of controlling the slider with your eyes while also wanting to check whether the object is at the correct location.</span></span> <span data-ttu-id="c4653-174">对于滑块的示例，最好将眼睛与手势结合使用。</span><span class="sxs-lookup"><span data-stu-id="c4653-174">For the example of the slider, it makes more sense to use eye-gaze in combination with hand gestures.</span></span> <span data-ttu-id="c4653-175">这意味着用户可以在多个滑杆之间快速轻松地进行切换，并收缩其拇指和食指来抓住和移动它。</span><span class="sxs-lookup"><span data-stu-id="c4653-175">This means that the user could quickly and effortlessly switch among a number of sliders, raising up their hand and pinching their thumb and index finger to grab and move it.</span></span> <span data-ttu-id="c4653-176">当释放了该收缩时，滑块将停止移动。</span><span class="sxs-lookup"><span data-stu-id="c4653-176">When the pinch is released, the slider stops moving.</span></span> <span data-ttu-id="c4653-177">简而言之，用户可能会被淹没和分散注意力，尤其是在信号对于该用户不精确的情况下。</span><span class="sxs-lookup"><span data-stu-id="c4653-177">In a nutshell, users could become overwhelmed and distracted, especially if the signal is imprecise for that user.</span></span> 
  
2. <span data-ttu-id="c4653-178">将**眼睛与其他输入组合在一起：** 将眼睛跟踪与其他输入（如手势、语音命令或按钮按下）的集成具有以下优点：</span><span class="sxs-lookup"><span data-stu-id="c4653-178">**Combine eye-gaze with other inputs:** The integration of eye tracking with other inputs, such as hand gestures, voice commands or button presses, provides several advantages:</span></span>
    - <span data-ttu-id="c4653-179">**允许免费观察：** 考虑到我们的主要角色是观察我们的环境，在不触发任何（视觉、听觉等）反馈或操作的情况下，允许用户进行查找，这一点非常重要。</span><span class="sxs-lookup"><span data-stu-id="c4653-179">**Allow for free observation:** Given that the main role of our eyes is to observe our environment, it is important that users are allowed to look around without triggering any (visual, auditory, etc.) feedback or actions.</span></span> 
    <span data-ttu-id="c4653-180">将目视跟踪与其他输入控件结合起来允许在目视跟踪观察和输入控制模式之间平稳过渡。</span><span class="sxs-lookup"><span data-stu-id="c4653-180">Combining eye tracking with another input control allows smooth transitioning between eye tracking observation and input control modes.</span></span>
  
    - <span data-ttu-id="c4653-181">**强大的上下文提供程序：** 使用有关用户在查找语音命令或执行手动手势的位置和内容的信息，可以在排列的情况下无缝地显示输入。</span><span class="sxs-lookup"><span data-stu-id="c4653-181">**Powerful context provider:** Using information about where and what the user is looking at while saying a voice command or performing a hand gesture allows seamlessly channeling the input across the field-of-view.</span></span> <span data-ttu-id="c4653-182">例如：说 _"将其放在该处"_ ，流畅地选择并在场景中放置一个全息图，只需查看目标及其目标位置即可。</span><span class="sxs-lookup"><span data-stu-id="c4653-182">For example: Say _"put that there"_ to quickly and fluently select and position a hologram across the scene by simply looking at a target and its intended destination.</span></span> 

    - <span data-ttu-id="c4653-183">**同步各模输入需要：** 将快速目视运动与更复杂的附加输入（例如长时间的语音命令或手势）相结合，可以满足用户在完成和识别其他输入命令之前的持续时间。</span><span class="sxs-lookup"><span data-stu-id="c4653-183">**Need for synchronizing multimodal inputs:** Combining rapid eye movements with more complex additional inputs, such as long voice commands or hand gestures, bears the risk that the user already continues to look around before the additional input command is finished and recognized.</span></span> <span data-ttu-id="c4653-184">因此，如果您创建自己的输入控件（例如，自定义笔势），请确保记录此输入或大致持续时间的开始，以将其与用户过去查看的内容相关联。</span><span class="sxs-lookup"><span data-stu-id="c4653-184">Hence, if you create your own input controls (e.g., custom hand gestures), make sure to log the onset of this input or approximate duration to correlate it with what a user had looked at in the past.</span></span>
    
3. <span data-ttu-id="c4653-185">**目视跟踪输入的细微反馈：** 当查看目标以指示系统正在按预期工作但应保持微妙时，提供反馈很有用。</span><span class="sxs-lookup"><span data-stu-id="c4653-185">**Subtle feedback for eye tracking input:** It's useful to provide feedback when a target is looked at to indicate that the system is working as intended but should be kept subtle.</span></span> <span data-ttu-id="c4653-186">这可能包括缓慢混合、放大和缩小、视觉对象突出显示或执行其他微妙目标行为（如缓慢增加目标大小），以指示系统正确检测到用户正在查看目标，而不会中断用户的当前工作流。</span><span class="sxs-lookup"><span data-stu-id="c4653-186">This can include slowly blending, in and out, visual highlights or perform other subtle target behaviors, such as slow motions, such as slightly increasing the target size, to indicate that the system correctly detected that the user is looking at a target without unnecessarily interrupting the user’s current workflow.</span></span> 

4. <span data-ttu-id="c4653-187">**避免将非自然目视运动作为输入进行：** 不要强制用户执行特定的目视运动（注视手势）来触发应用程序中的操作。</span><span class="sxs-lookup"><span data-stu-id="c4653-187">**Avoid enforcing unnatural eye movements as input:** Do not force users to perform specific eye movements (gaze gestures) to trigger actions in your application.</span></span>

5. <span data-ttu-id="c4653-188">**帐户 imprecisions：** 我们区分对用户显而易见的两种类型的 imprecisions：偏移和抖动。</span><span class="sxs-lookup"><span data-stu-id="c4653-188">**Account for imprecisions:** We distinguish two types of imprecisions which are noticeable to users: offset and jitter.</span></span> <span data-ttu-id="c4653-189">解决偏移量的最简单方法是提供足够大的目标来与进行交互。</span><span class="sxs-lookup"><span data-stu-id="c4653-189">The easiest way to address an offset is to provide sufficiently large targets to interact with.</span></span> <span data-ttu-id="c4653-190">建议使用大于2°的视觉角度作为参考。</span><span class="sxs-lookup"><span data-stu-id="c4653-190">It is suggested that you use a visual angle greater than 2° as a reference.</span></span> <span data-ttu-id="c4653-191">例如，当您拉伸 arm 时，缩略图约为2°。</span><span class="sxs-lookup"><span data-stu-id="c4653-191">For instance, your thumbnail is about 2° in visual angle when you stretch out your arm.</span></span> <span data-ttu-id="c4653-192">因此，我们的指导如下：</span><span class="sxs-lookup"><span data-stu-id="c4653-192">This leads to the following guidance:</span></span>
    - <span data-ttu-id="c4653-193">不要强制用户选择 "小目标"。</span><span class="sxs-lookup"><span data-stu-id="c4653-193">Do not force users to select tiny targets.</span></span> <span data-ttu-id="c4653-194">研究表明，如果目标足够大，并且系统设计良好，则用户会将其交互描述为轻松和神奇。</span><span class="sxs-lookup"><span data-stu-id="c4653-194">Research has shown that if targets are sufficiently large, and that the system is designed well, users describe their interactions as effortless and magical.</span></span> <span data-ttu-id="c4653-195">如果目标太小，则用户就会将体验描述为费力、令人沮丧。</span><span class="sxs-lookup"><span data-stu-id="c4653-195">If targets become too small, users describe the experience as fatiguing and frustrating.</span></span>
